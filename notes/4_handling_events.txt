}

it's a good idea to encapsulate the event handler into its own function, and it's common practice to name the function handle + the event name – in our case handleClick.

const handleClick = () => {}

const handleFocus = () => {}

const handleChange = () => {}

Next, you'll notice that we're passing our event handler as a reference to onClick, not as an invocation. Note the difference between these two onClick props.

do this
    <button onClick={handleClick}>
    Passing a Reference
    </button>

not this 
    <button onClick={handleClick()}>
    Passing an Invocation
    </button>

you'll notice that we created our event handler inside the component, not outside of it. This is also a common pattern, but it may feel unintuitive at first. 

By declaring our handleClick function inside of our component, we automatically have access to the message prop via JavaScript's normal lexical scoping rules.
